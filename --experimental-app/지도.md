# 지도

프로젝트에서 지도는 두 가지 용도로 사용됩니다.

| 기능 | 컴포넌트 | 라이브러리 | 목적 |
|------|----------|------------|------|
| 여행지 지도 | `TravelMap.tsx` | react-leaflet | 저장된 여행지 좌표 시각화 |
| 위치 선택 | `KakaoMapPicker.tsx` | 카카오맵 SDK | 목적지 추가 시 좌표 검색/선택 |

---

## 1. TravelMap — 여행지 시각화

**파일:** `components/dashboard/TravelMap.tsx`

### 특징
- `"use client"` 컴포넌트
- `dashboardClient.tsx`에서 `dynamic(() => import(...), { ssr: false })` 로 로드 (leaflet SSR 문제 방지)
- **좌표(lat/lng)가 있는 목적지만** 마커로 표시 — 좌표 없으면 컴포넌트 자체 미렌더링
- 지도 중심: 전체 마커의 평균 좌표로 자동 계산

### 마커
- 모든 필터링된 여행의 coordinates를 마커로 표시
- 팝업: 목적지 이름 / 여행 제목 / 카테고리 / 비용

### Polyline
- 같은 여행 내 목적지들을 점선으로 연결 (`dashArray: "6"`)
- 색상: `#8b5cf6` (보라색)
- 마커가 2개 이상인 여행에만 적용

### 아이콘 이슈 해결
```typescript
// Next.js 번들 경로 문제 → CDN URL로 교체
L.Icon.Default.mergeOptions({
  iconUrl: "https://unpkg.com/leaflet@1.9.4/dist/images/marker-icon.png",
  ...
});
```
`FixLeafletIcons` 컴포넌트를 MapContainer 내부에서 `useEffect`로 실행.

### 사용 라이브러리
- `react-leaflet` v5.0.0
- `leaflet` v1.9.4
- `@types/leaflet`
- 지도 타일: OpenStreetMap

## 2. KakaoMapPicker — 위치 선택기

**파일:** `components/dashboard/travelForm/KakaoMapPicker.tsx`

### 특징
- `"use client"` 컴포넌트
- `DestinationSection.tsx`에서 `dynamic(() => import(...), { ssr: false })` 로 로드
- 카카오맵 SDK를 `document.head`에 동적으로 스크립트 삽입 (중복 삽입 방지 로직 포함)

### 카카오맵 SDK 로드 방식
```
appkey: NEXT_PUBLIC_KAKAO_MAP_KEY
libraries: services (장소 검색용)
autoload: false (수동 초기화)
```

스크립트 로드 후 반드시 `kakao.maps.load(callback)` 콜백 안에서 지도를 초기화해야 함.

```typescript
// 스크립트가 이미 있을 때 (재마운트)
window.kakao.maps.load(() => { initMap(); });

// 스크립트를 새로 추가할 때
script.onload = () => {
  window.kakao.maps.load(() => { initMap(); });
};
```

### ⚠️ 트러블슈팅 — 지도 미로드 (2026-04-27)

**증상:** "지도에서 위치 선택" 버튼을 눌러도 지도가 뜨지 않고 "지도 로딩 중..." 상태에서 멈춤

**원인:** 카카오맵 SDK는 스크립트 `onload` 시점에 `window.kakao.maps`는 생성되지만, `Map`·`LatLng` 등 실제 클래스는 **내부 비동기 초기화** 이후에 준비됨. 기존 코드는 `window.kakao.maps.Map` 존재 여부를 100ms 폴링으로 확인했으나, 이 방식으로는 초기화 완료를 감지하지 못해 10초 후 타임아웃 처리됨.

| 항목 | 기존 (오류) | 수정 후 |
|------|------------|---------|
| 스크립트 URL | `autoload` 파라미터 없음 | `autoload=false` 추가 |
| 초기화 감지 | `Map` 클래스 존재 여부 폴링 | `kakao.maps.load(callback)` 콜백 사용 |
| 재마운트 처리 | 폴링 재시도 | `window.kakao.maps` 있으면 바로 `load()` 호출 |

**핵심 원칙:** 카카오맵 SDK는 `kakao.maps.load()` 콜백 내부에서만 지도 객체를 안전하게 생성할 수 있음.

### 장소 검색 흐름
```
사용자 입력 → keywordSearch() → 최대 6개 결과 표시
→ 결과 클릭 → 지도 이동 + 마커 표시 + 콜백(lat, lng, placeName)
```

### 지도 직접 클릭 — 장소명 자동 조회
지도 클릭 시 좌표만 저장하는 게 아니라, 2단계로 장소명을 자동 조회해 목적지 이름 input에 채움.

```
클릭 → coord2Address(lng, lat) → 주소 문자열 획득
  → Places.keywordSearch(주소, { x, y, radius: 30, size: 1, sort: DISTANCE })
    → 결과 있음: places[0].place_name → placeMarker (가게명/아파트명 등)
    → 결과 없음: 주소 문자열 → placeMarker (도로명/지번 주소 fallback)
```

- 반경 30m 이내 등록 장소가 있으면 장소명(가게, 아파트 등) 우선 사용
- 산속·논밭 등 등록 장소가 없는 경우 도로명 주소로 fallback
- `placeName`이 빈 문자열이면 `DestinationSection`에서 `SET_DEST_NAME` dispatch 생략 (좌표만 저장)

### 초기값 처리
- `initialLat`, `initialLng` props 있으면 해당 위치로 지도 중심 이동 + 마커 표시 (수정 시 사용)

### 콜백
```typescript
onSelect: (lat: number, lng: number, placeName: string) => void
```
→ `DestinationSection`에서 받아 `dispatch(SET_DEST_LAT)`, `dispatch(SET_DEST_LNG)`, `dispatch(SET_DEST_PLACE_NAME)` 실행
→ `placeName`이 있으면 추가로 `dispatch(SET_DEST_NAME)` 실행 → 목적지 이름 input 자동 입력

### 기본 지도 설정
- 중심: 한국 중심 (36.5, 127.5)
- 줌 레벨: 8 (초기) / 4 (장소 선택 후)

## 좌표 데이터 흐름

```
KakaoMapPicker (선택)
  → DestinationSection dispatch (SET_DEST_LAT / SET_DEST_LNG)
  → ADD_DESTINATION → destinations[] 배열에 lat/lng 포함
  → travelModal INSERT destinations (lat, lng 컬럼)
  → Supabase destinations 테이블 저장
  → dashboard/page.tsx 쿼리에서 lat/lng 포함 패칭
  → TravelMap에서 마커로 시각화
```
