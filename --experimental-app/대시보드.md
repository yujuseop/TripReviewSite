# 대시보드

## 파일 구조

```
app/dashboard/page.tsx              ← 서버 컴포넌트 (데이터 패칭)
app/dashboard/dashboardClient.tsx   ← 클라이언트 컴포넌트 (UI)
hooks/useDashboard.ts               ← 대시보드 전체 상태 관리
components/dashboard/
  ├── TravelList.tsx                ← 여행 목록
  ├── ImageList.tsx                 ← 이미지 슬라이더
  ├── TotalCostDisplay.tsx          ← 총 여행 경비
  ├── TravelMap.tsx                 ← 여행지 지도 (react-leaflet)
  ├── travelModal.tsx               ← 여행 추가 모달
  ├── TravelEditModal.tsx           ← 여행 수정 모달
  └── detail/
      ├── TravelDetailModal.tsx     ← 여행 상세 모달
      ├── TimelineSection.tsx       ← 일차별 목적지 타임라인
      └── ReviewSection.tsx         ← 리뷰/무드태그/사진
```

---

## 데이터 패칭 (page.tsx)

서버 컴포넌트에서 Supabase 쿼리로 초기 데이터를 가져와 `DashboardClient`에 props로 전달.

**패칭 항목:**
- `profiles` 테이블 — 닉네임, 역할
- `trip` 테이블 — 전체 여행 목록 (최신순)
  - `destinations` — id, name, description, day, order_num, **category, lat, lng, cost**
  - `reviews` — id, content, rating, created_at, user_id, images, **mood_tags, one_line_summary**

## useDashboard 훅

**상태 목록:**

| 상태 | 타입 | 역할 |
|------|------|------|
| `date` | `Date \| null` | 캘린더 선택 날짜 |
| `travels` | `Travel[]` | 전체 여행 목록 (원본) |
| `filteredTravels` | `Travel[]` | 필터 적용된 목록 (useMemo) |
| `activeFilter` | `FilterType` | 현재 필터 |
| `selectedReview` | `Review \| null` | 상세/수정 모달용 선택 리뷰 |
| `selectedTravel` | `Travel \| null` | 상세/수정/삭제 모달용 선택 여행 |

**핸들러 목록:**

| 핸들러 | 동작 |
|--------|------|
| `handleTravelClick` | `selectedTravel` 설정 → 상세 모달 오픈 |
| `handleTravelEditOpen` | `selectedTravel` 설정 → 수정 모달 오픈 |
| `handleTravelUpdated` | 수정 완료 후 `travels` 배열 + `selectedTravel` 동시 갱신 |
| `handleTravelDeleteClick` | `selectedTravel` 설정 → 삭제 확인 모달 오픈 |
| `handleReviewClick` | `selectedReview` 설정 → 리뷰 상세 모달 오픈 |
| `handleReviewEditClick` | `selectedReview` 설정 → 리뷰 수정 모달 오픈 |

## 필터 기능

**필터 종류:** `전체` / `맛집` / `관광지` / `평점순`

| 필터 | 동작 |
|------|------|
| 전체 | 모든 여행 표시 |
| 맛집 | destinations 중 category === "맛집" 인 여행만 |
| 관광지 | destinations 중 category === "관광지" 인 여행만 |
| 평점순 | 리뷰 평균 평점 높은 순 정렬 |

UI: 대시보드 "내 여행 목록" 오른쪽에 pill 형태 탭으로 표시.

## TotalCostDisplay

- `travels` props로 받아서 모든 destinations의 `cost` 합산
- 합계가 0이면 렌더링 자체를 하지 않음
- 표시 형식: `총 여행 경비 / 123,000원`

## 여행 상세 모달 (TravelDetailModal)

**파일:** `components/dashboard/detail/TravelDetailModal.tsx`

**구성:**
- 상단: 여행 날짜 + "수정" 버튼 + `TotalCostDisplay` (해당 여행만)
- 중단: `TimelineSection` — destinations를 day별 그룹화, 카테고리 색상 뱃지 + 비용 표시
- 하단: `ReviewSection` — 평점, 한줄평, 무드태그 색상 뱃지, 리뷰 본문, 사진

**TotalCostDisplay 재사용:** `travels={[selectedTravel]}` 로 단일 여행 경비만 표시.

## 여행 수정 모달 (TravelEditModal)

**파일:** `components/dashboard/TravelEditModal.tsx`

**편집 가능 항목:**
- 여행 제목 (input pre-fill)
- 기존 목적지 목록 (✕ 버튼으로 삭제 예약, 저장 시 DB DELETE)
- 새 목적지 추가 (`DestinationSection` 재사용 — 카카오맵 포함)

**저장 시 DB 처리 순서:**
1. 제목 변경 시 `trip` 테이블 UPDATE
2. 삭제된 목적지 ID 배열로 `destinations` 테이블 DELETE
3. 새로 추가된 목적지 `destinations` 테이블 INSERT

**저장 완료 후:** `handleTravelUpdated` 호출 → `travels` 배열과 `selectedTravel` 즉시 갱신 (재패칭 없음)
